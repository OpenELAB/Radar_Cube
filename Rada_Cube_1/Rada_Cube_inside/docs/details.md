# FeedbackController 视听反馈事件定义

详细设计原则见 [design.md](design.md)。

## 场景 - 事件 Event

每个事件统一描述以下内容：

- **触发条件**：业务层在什么条件下通知 FeedbackController。
- **状态变化**：FeedbackController 需要保存或更新的状态。
- **灯效**：颜色、动态效果和 `LedZone`，或者不受 `LedZone` 限制的全局动画。
- **声音**：距离蜂鸣、短提示音或最终语音。
- **流程**：业务流程是否等待声音或动画结束。

文档中的“声音顺序”和“流程等待”是两个不同概念：

- **声音顺序播放**：新声音在音频调度器中等待前一个声音播放完成。此时 `main` 不等待，仍继续接收 ACK、ESP-NOW 数据和处理系统任务。
- **流程不等待**：`main` 不因该声音停下业务流程；不代表新声音必须打断前一个声音。
- **流程等待**：`main` 暂停进入下一个业务阶段，直到指定音频序列完成或超时，但仍必须运行音频更新、无线接收和系统维护任务。

除特别说明外，所有事件的业务流程均不等待。任何流程等待都必须设置超时。

### 通用事件

PS：等待/不等待可以进一步讨论和后续修改

#### Event：系统上电

- **触发条件**：主机完成必要硬件初始化，可以开始用户反馈。
- **状态变化**：清空旧的临时灯效和音频状态。
- **灯效**：四灯执行一次顺时针启动跑马灯，颜色和具体动画由产品设计确定。
- **声音**：可播放一次简短启动音 `sys_boot.wav`，不播放冗长语音。
- **流程**：可以等待启动动画和启动音结束，但必须设置超时；结束后根据配对状态进入对应模式灯效。

#### Event：进入休眠或关机

- **触发条件**：工作结束、超时、用户按键退出，或者系统准备进入深度睡眠。
- **状态变化**：停止距离反馈，清理不再有意义的普通提示。
- **灯效**：四灯执行一次逆时针退场动画并逐渐熄灭。
- **声音**：播放一次关机提示音或简短语音 `sys_shutdown.wav`。
- **流程**：等待最终提示结束后再休眠；必须设置最大等待时间。

#### Event：主机普通低电

- **触发条件**：主机电量低于普通低电阈值，但仍可继续工作。
- **状态变化**：记录主机低电状态；同一低电状态只产生一次提示。
- **灯效**：不覆盖正在显示的距离灯效；在非工作模式或距离反馈结束后，可显示一次全环琥珀色短脉冲。
- **声音**：播放一次简短注意音；工作期间不得用长语音打断距离蜂鸣。
- **流程**：不等待；需要防抖和冷却，电量未恢复前不得重复提示。

#### Event：主机严重低电

- **触发条件**：主机电量不足以可靠继续工作，即将退出或休眠。
- **状态变化**：标记系统即将停止工作，停止接受新的普通提示。
- **灯效**：全环琥珀色或红色严重低电动画，具体效果由产品设计确定。
- **声音**：停止距离蜂鸣，播放一次需要用户行动的严重低电语音 `power_critical.wav`。
- **流程**：等待语音结束或超时，然后退出工作并进入休眠。

#### Event：严重系统故障

- **触发条件**：出现无法归类或无法恢复的硬件、音频、存储或系统异常，设备不能可靠工作。
- **状态变化**：进入系统不可用状态，取消普通提示和距离反馈。
- **灯效**：全环红色故障闪烁，持续到退出、复位或休眠。
- **声音**：播放一次简短故障音或系统故障语音 `fault_system.wav`；必要时周期性播放简短故障音。
- **流程**：不因声音阻塞故障处理；如果必须关机，按关机流程执行。

### UNPAIRED_MODE

#### Event：检测到设备未配对

- **触发条件**：系统启动后没有完整、有效的左右设备配对信息，或者配对信息已被清除。
- **状态变化**：`LedZone = None`，系统保持未配对状态。
- **灯效**：默认熄灭；如果需要提醒用户，可全环蓝色短呼吸或短脉冲一次。
- **声音**：可播放一次简短未配对提示 `mode_unpaired.wav`，不得反复播放。
- **流程**：不自动无限重试配对；等待用户进入配对模式，或者按现有业务流程进入休眠。

### PAIRED_MODE

> 当前代码中的 `PAIRED_MODE` 实际表示“配对模式”，本文沿用现有名称。

#### Event：进入配对模式

- **触发条件**：用户长按指定按键进入配对流程。
- **状态变化**：清空本次配对结果，`LedZone = None`。
- **灯效**：四灯蓝色呼吸，表示正在等待设备配对。
- **声音**：可播放简短的进入配对提示 `mode_pairing.wav`。
- **流程**：无需等待语音完成才开始无线配对；配对流程可以同时进行。

#### Event：左侧首先配对成功

- **触发条件**：左侧设备完成配对，右侧尚未完成。
- **状态变化**：`LedZone = Left`。
- **灯效**：结束四灯蓝色呼吸；左半环绿色脉冲一次，随后左半环绿色常亮，右半环熄灭。
- **声音**：播放一次短促上升音 `tone_success_up.wav`。
- **流程**：不等待，继续等待右侧设备。

#### Event：右侧首先配对成功

- **触发条件**：右侧设备完成配对，左侧尚未完成。
- **状态变化**：`LedZone = Right`。
- **灯效**：结束四灯蓝色呼吸；右半环绿色脉冲一次，随后右半环绿色常亮，左半环熄灭。
- **声音**：播放一次短促上升音 `tone_success_up.wav`。
- **流程**：不等待，继续等待左侧设备。

#### Event：双侧配对成功

- **触发条件**：左右设备均完成配对。
- **状态变化**：`LedZone = Both`，保存完整配对信息。
- **灯效**：四灯执行绿色完成动画，结束后四灯绿色常亮。
- **声音**：播放一次最终配对成功提示 `pair_ok_both.wav`。
- **流程**：等待最终提示结束或超时，然后完成配对流程。
- **合并规则**：第二侧成功直接导致双侧完成时，可省略第二次单侧上升音，避免“上升音 + 上升音 + 完成语音”过密。

#### Event：配对超时，仅左侧成功

- **触发条件**：配对窗口结束，左侧成功、右侧失败。
- **状态变化**：保留左侧结果，`LedZone = Left`；系统仍属于未完整配对。
- **灯效**：右半环红色短脉冲或双闪一次，随后恢复左半环绿色常亮、右半环熄灭。
- **声音**：只播放一次最终结果 `pair_fail_right.wav`，不再播放独立的右侧失败音和“请重新配对”音。
- **流程**：等待最终提示结束或超时，然后退出配对流程。

#### Event：配对超时，仅右侧成功

- **触发条件**：配对窗口结束，右侧成功、左侧失败。
- **状态变化**：保留右侧结果，`LedZone = Right`；系统仍属于未完整配对。
- **灯效**：左半环红色短脉冲或双闪一次，随后恢复右半环绿色常亮、左半环熄灭。
- **声音**：只播放一次最终结果 `pair_fail_left.wav`。
- **流程**：等待最终提示结束或超时，然后退出配对流程。

#### Event：配对超时，双侧均失败

- **触发条件**：配对窗口结束，左右设备均未成功。
- **状态变化**：`LedZone = None`，不保存不完整或无效配对信息。
- **灯效**：四灯红色双闪或失败动画，结束后熄灭。
- **声音**：只播放一次最终结果 `pair_fail_both.wav`。
- **流程**：等待最终提示结束或超时，然后返回未配对状态。

### WORK_MODE

#### Event：开始唤醒外部设备

- **触发条件**：系统进入工作模式并开始通过 LoRa 唤醒左右设备。
- **状态变化**：清空本次唤醒结果，`LedZone = None`。
- **灯效**：显示统一的工作准备灯效，具体颜色和动态由产品设计确定。
- **声音**：可播放很短的开始提示音，不使用会延迟距离反馈的长语音。
- **流程**：不等待，立即开始唤醒和接收确认。

#### Event：左侧首先唤醒成功

- **触发条件**：左侧返回有效唤醒确认，右侧尚未确认。
- **状态变化**：`LedZone = Left`。
- **灯效**：左半环绿色脉冲一次，随后保持左半环工作准备状态。
- **声音**：把一次短促上升音加入本次唤醒反馈序列；如果前面有声音，则按顺序等待前一个声音结束。
- **流程**：`main` 不等待该上升音，继续唤醒右侧并接收确认。

#### Event：右侧首先唤醒成功

- **触发条件**：右侧返回有效唤醒确认，左侧尚未确认。
- **状态变化**：`LedZone = Right`。
- **灯效**：右半环绿色脉冲一次，随后保持右半环工作准备状态。
- **声音**：把一次短促上升音加入本次唤醒反馈序列；如果前面有声音，则按顺序等待前一个声音结束。
- **流程**：`main` 不等待该上升音，继续唤醒左侧并接收确认。

#### Event：双侧唤醒成功

- **触发条件**：左右设备均返回有效唤醒确认。
- **状态变化**：`LedZone = Both`，进入“等待唤醒完成反馈”状态；期间收到的最新有效距离可以缓存，但暂不输出距离视听反馈。
- **灯效**：四灯绿色短脉冲，随后保持四灯绿色工作准备状态。
- **声音**：第二侧成功时也必须加入一次上升音；两个单侧上升音按 ACK 到达顺序完整播放，之后播放一次双侧唤醒成功语音 `wake_ok.wav`。
- **流程**：`main` 等待整个唤醒反馈序列结束或超时，然后才进入距离反馈阶段。等待期间继续接收无线数据、更新音频和缓存最新距离。
- **合并规则**：左右 ACK 即使接近同时到达，也保留两个上升音，不合并；最终成功语音只播放一次。

#### Event：唤醒超时，仅一侧成功

- **触发条件**：达到最大唤醒重试次数，只有一侧返回确认。
- **状态变化**：记录单侧可用状态；是否允许降级进入工作模式由业务需求决定。
- **灯效**：成功侧保持工作准备状态，失败侧熄灭或显示一次失败脉冲。
- **声音**：播放一次短促下降音。
- **流程**：如果允许单侧工作，立即以该侧进入工作模式；如果不允许，则发送结束帧并退出。

#### Event：唤醒超时，双侧均失败

- **触发条件**：达到最大唤醒重试次数，左右设备均未确认。
- **状态变化**：`LedZone = None`，系统不可进入距离工作状态。
- **灯效**：四灯显示系统不可用的全局故障效果。
- **声音**：播放一次简短双侧失败音，不启动距离蜂鸣。
- **流程**：发送结束帧，退出工作模式，不等待冗长语音。

#### Event：收到第一帧有效距离

- **触发条件**：至少一侧在线设备返回有效距离数据，并且双侧唤醒成功反馈序列已经播放完成或等待超时。
- **状态变化**：进入距离反馈状态；使用在线设备距离的最小值。
- **灯效**：停止工作准备动画，显示当前缓存的最新距离对应灯效。
- **声音**：根据当前缓存的最新最小距离启动蜂鸣或保持静音。
- **流程**：不等待。

#### Event：距离等级变化

- **触发条件**：经过迟滞和最短保持时间后，最小有效距离进入新的距离等级。
- **状态变化**：更新当前距离等级。
- **灯效**：把新距离等级对应的统一灯效应用到当前在线 `LedZone`。
- **声音**：替换为新距离等级对应的蜂鸣；如果距离进入安全区，则停止蜂鸣。
- **流程**：不等待，不进入普通提示队列。

建议映射：

| 距离状态 | 在线区域灯效 | 距离声音 |
| --- | --- | --- |
| 安全距离 | 绿色常亮 | 静音 |
| 较远距离 | 绿色慢闪 | 慢速蜂鸣 |
| 中距离 | 黄色闪烁 | 中速蜂鸣 |
| 近距离 | 红色快闪 | 快速蜂鸣 |
| 危险距离 | 红色常亮 | 连续或最高频蜂鸣 |

> 注意：以上颜色、距离等级、动态模式和蜂鸣节奏仅供设计讨论参考，可能与当前代码不一致。最终方案应根据实际产品设计、距离阈值、外壳柔光效果和实机用户测试确定。

#### Event：左侧确认失联

- **触发条件**：左侧超过失联阈值未收到有效数据，并通过防抖确认；右侧仍在线。
- **状态变化**：从在线区域移除左侧，`LedZone = Right`。
- **灯效**：左半环熄灭，右半环继续显示当前最小有效距离灯效。
- **声音**：暂停距离蜂鸣，播放一次 200 至 500 ms 的下降音，然后根据右侧最新距离重新选择蜂鸣。
- **流程**：不等待，不播放长语音；同一失联状态只提示一次。

#### Event：右侧确认失联

- **触发条件**：右侧超过失联阈值未收到有效数据，并通过防抖确认；左侧仍在线。
- **状态变化**：从在线区域移除右侧，`LedZone = Left`。
- **灯效**：右半环熄灭，左半环继续显示当前最小有效距离灯效。
- **声音**：暂停距离蜂鸣，播放一次下降音，然后根据左侧最新距离重新选择蜂鸣。
- **流程**：不等待，不播放长语音；同一失联状态只提示一次。

#### Event：双侧确认失联

- **触发条件**：左右两侧均达到失联条件，或者第二侧在短时间聚合窗口内也确认失联。
- **状态变化**：`LedZone = None`，当前距离无效，停止距离反馈。
- **灯效**：四灯进入系统不可用的全局琥珀色或红色故障效果，不能仅保持全灭。
- **声音**：停止距离蜂鸣，播放一次比单侧下降音更明确的双侧故障短音。
- **流程**：不等待；如果继续停留在工作模式，可按产品测试结果周期性播放简短不可用提示。
- **合并规则**：若左右失联在短时间内连续发生，只播放双侧故障音，不连续播放两次单侧下降音。

#### Event：左侧恢复在线

- **触发条件**：左侧失联后连续收到规定数量的有效数据，右侧仍在线。
- **状态变化**：将左侧重新加入在线区域，`LedZone = Both`。
- **灯效**：左半环绿色短脉冲一次，然后四灯显示当前最小距离灯效。
- **声音**：可播放一次很短的上升音；如提示过密，可只使用灯效。
- **流程**：不等待；提示结束后根据左右最新最小距离重新选择蜂鸣。

#### Event：右侧恢复在线

- **触发条件**：右侧失联后连续收到规定数量的有效数据，左侧仍在线。
- **状态变化**：将右侧重新加入在线区域，`LedZone = Both`。
- **灯效**：右半环绿色短脉冲一次，然后四灯显示当前最小距离灯效。
- **声音**：可播放一次很短的上升音；如提示过密，可只使用灯效。
- **流程**：不等待；提示结束后根据左右最新最小距离重新选择蜂鸣。

#### Event：双侧从不可用状态恢复

- **触发条件**：系统原本双侧失联，至少一侧重新连续返回有效数据。
- **状态变化**：`LedZone` 更新为实际恢复的 `Left`、`Right` 或 `Both`。
- **灯效**：恢复区域绿色脉冲一次，随后立即显示当前距离灯效。
- **声音**：播放一次恢复上升音，不为左右两侧连续播放多次。
- **流程**：不等待，立即恢复距离反馈。

#### Event：单侧距离数据持续异常

- **触发条件**：设备仍有通信，但连续返回无效距离、协议异常或传感器故障数据，达到故障阈值。
- **状态变化**：故障侧暂时从可用于距离计算的区域移除；另一侧正常时继续降级工作。
- **灯效**：故障侧执行一次红色短脉冲后熄灭；可用侧继续显示距离灯效。
- **声音**：播放一次短促故障下降音，然后根据剩余有效距离恢复蜂鸣。
- **流程**：不等待；使用故障冷却时间，不能每收到一帧无效数据都提示。

#### Event：双侧距离数据持续异常

- **触发条件**：左右两侧均达到数据异常或传感器故障阈值。
- **状态变化**：`LedZone = None`，停止使用当前距离。
- **灯效**：四灯进入全局红色故障效果。
- **声音**：停止距离蜂鸣，播放一次双侧故障短音。
- **流程**：不等待；有效数据恢复并通过确认后才能重新开始距离反馈。

#### Event：单侧设备普通低电

- **触发条件**：在线设备上报普通低电状态。
- **状态变化**：记录对应侧低电状态；不影响其距离有效性。
- **灯效**：不覆盖工作中的距离灯效；可在工作结束后或安全距离状态下显示一次对应侧琥珀色短脉冲。
- **声音**：播放一次简短注意音，或者仅使用灯效；不得播放长语音。
- **流程**：不等待；同一低电状态只提示一次。

#### Event：单侧设备严重低电或即将离线

- **触发条件**：外部设备上报严重低电，预计无法继续可靠提供距离。
- **状态变化**：标记对应侧即将不可用；实际停止有效数据后按失联或故障事件处理。
- **灯效**：对应侧琥珀色或红色短脉冲一次，随后仍优先显示有效距离。
- **声音**：播放一次较明确但简短的警告音。
- **流程**：不等待，不在数据仍有效时提前删除该侧。

#### Event：退出工作模式

- **触发条件**：工作超时、用户按键、收到结束帧或上层要求退出。
- **状态变化**：停止距离蜂鸣，清理距离等级和工作期间的临时提示。
- **灯效**：停止距离灯效，随后执行休眠或关机事件。
- **声音**：不额外播放重复的“工作结束”语音，由关机/休眠提示统一负责。
- **流程**：通知外部设备结束工作后进入休眠流程。
