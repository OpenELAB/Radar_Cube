# FeedbackController 事件定义

详细原则见 [design.md](design.md)。本文件只定义第一版核心事件，不展开低电、系统故障、测试模式、配置模式和恢复出厂模式。

## 通用规则

- 所有 `onXxxEvent()` 都立即返回，不在函数内部等待声音或动画结束。
- main 想完整保留前一个反馈时，先等 `FeedbackController` 空闲或超时；想抢占时，直接调用下一个事件。
- 除倒车距离显示外，新事件可以替换或打断前一个视听反馈。
- 倒车距离显示期间不插入长语音，只允许短音效短暂打断距离蜂鸣。
- `playOnce()` 打断 `playLoop()` 后，播放完成自动恢复 loop；这个语义由 SpeakerController 负责。
- AudioId 到路径的映射和倒车期间黑名单判断放在 AudioCatalog；SpeakerController 只接收路径并播放，不判断业务语义。
- 是否处于倒车距离显示状态由 FeedbackController 维护；该状态下只允许短提示音插入。
- 需要顺序播放的场景由 main 控制调用时机，例如 main 等左上升音结束后再触发右上升音，再触发 `wake_ok.wav`。

## 通用事件

### Event：系统上电

- **接口**：`onSystemBootEvent(startup_scene)`
- **触发条件**：主机完成必要硬件初始化。
- **参数**：`startup_scene` 表示上电后预计进入的场景，例如 `Pairing` 或 `Working`。
- **反馈**：播放上电跑马灯和简短启动音；配对场景可衔接配对提示，工作场景可被后续唤醒/距离事件打断。
- **流程**：事件立即返回。main 是否等待启动反馈，由当前业务场景决定。

### Event：统一休眠或关机

- **接口**：`onShutdownEvent(current_scene)`
- **触发条件**：系统准备进入休眠或关机。
- **参数**：`current_scene` 表示当前从哪个场景退场。
- **反馈**：停止距离反馈，播放退场动画和 `sys_shutdown.wav`。
- **流程**：事件立即返回；main 通过 `isBusy()` 加超时等待最终提示后再休眠。

## 未配对

### Event：检测到设备未配对

- **接口**：`onUnpairedDetectedEvent()`
- **触发条件**：启动后没有完整有效的左右设备配对信息。
- **状态变化**：`SensorSet = None`。
- **反馈**：默认保持安静；如需提醒，可播放一次轻量蓝色提示或 `mode_unpaired.wav`。
- **流程**：等待用户进入配对模式，或按业务流程休眠。

## 配对模式

当前代码中的 `PAIRED_MODE` 实际表示“配对模式”，本文沿用现有名称。

### Event：进入配对模式

- **接口**：`onPairingStartedEvent()`
- **触发条件**：用户进入配对流程。
- **状态变化**：清空本次配对结果，`SensorSet = None`。
- **反馈**：四灯蓝色呼吸，可播放 `mode_pairing.wav`。
- **流程**：事件立即返回；main 可选择等待反馈结束后再发送配对信息。

### Event：单侧配对成功

- **接口**：`onPairLeftSucceededEvent()` / `onPairRightSucceededEvent()`
- **触发条件**：左侧或右侧设备完成配对，另一侧尚未完成。
- **状态变化**：`SensorSet = Left` 或 `Right`。
- **反馈**：成功侧单颗设备灯绿色常亮，播放一次短上升音。
- **流程**：不等待，继续等待另一侧。

### Event：双侧配对成功

- **接口**：`onPairBothSucceededEvent()`
- **触发条件**：左右设备均完成配对。
- **状态变化**：`SensorSet = Both`，保存完整配对信息。
- **反馈**：四灯绿色完成动画，播放 `pair_ok_both.wav`。
- **流程**：事件立即返回；main 可等待 `isBusy()` 空闲或超时后结束配对流程。
- **说明**：第二侧成功直接导致双侧完成时，可省略第二次单侧上升音。

### Event：配对超时

- **接口**：`onPairingTimedOutEvent(paired_sensors)`
- **触发条件**：配对窗口结束，但双侧没有全部完成。
- **参数**：`paired_sensors` 只允许 `None`、`Left` 或 `Right`；`Both` 应走“双侧配对成功”。
- **反馈**：
  - `Left`：右侧失败提示，播放 `pair_fail_right.wav`。
  - `Right`：左侧失败提示，播放 `pair_fail_left.wav`。
  - `None`：四灯失败动画，播放 `pair_fail_both.wav`。
- **流程**：事件立即返回；main 可等待最终提示后退出配对流程。

## 工作模式

main 进入工作模式后直接唤醒外部设备，不需要单独通知“开始唤醒”事件。

### Event：单侧唤醒成功

- **接口**：`onWakeLeftSucceededEvent()` / `onWakeRightSucceededEvent()`
- **触发条件**：左侧或右侧返回有效唤醒确认。
- **状态变化**：更新唤醒阶段的 `SensorSet`。
- **反馈**：成功侧单颗设备灯绿色常亮，播放一次短上升音。
- **流程**：不等待，main 继续接收 ACK 和距离数据。
- **说明**：如果需要“左上升音、右上升音、wake_ok”的完整顺序，由 main 通过 `isBusy()` 控制调用时机。

### Event：唤醒超时

- **接口**：`onWakeTimedOutEvent(awake_sensors)`
- **触发条件**：达到最大唤醒重试次数，但双侧没有全部确认。
- **参数**：`awake_sensors` 只允许 `None`、`Left` 或 `Right`；`Both` 由两次单侧唤醒成功自然形成。
- **反馈**：
  - `Left` / `Right`：失败侧提示，播放短下降音。
  - `None`：四灯系统不可用提示，播放双侧失败短音。
- **流程**：是否允许单侧降级工作由 main 决定；双侧失败时退出工作流程。

### Event：距离等级更新

- **接口**：`onDistanceLevelChangedEvent(active_sensors, level)`
- **触发条件**：收到第一帧有效距离，或距离等级经迟滞确认后变化。
- **参数**：`active_sensors` 表示当前在线并可用于距离计算的设备；`level` 表示当前距离等级。
- **反馈**：根据 `active_sensors` 选择灯区，根据 `level` 选择距离灯效和蜂鸣。
- **流程**：不等待；如果业务上需要先播放 `wake_ok.wav`，main 自己等待后再开始上报距离等级。

参考映射：

| 距离状态 | 距离灯效 | 距离声音 |
| --- | --- | --- |
| 安全距离 | 绿色常亮 | 静音 |
| 较远距离 | 绿色慢闪 | 慢速蜂鸣 |
| 中距离 | 黄色闪烁 | 中速蜂鸣 |
| 近距离 | 红色快闪 | 快速蜂鸣 |
| 危险距离 | 红色常亮 | 连续或最高频蜂鸣 |

以上颜色、距离等级和蜂鸣节奏仅供第一版设计参考，最终以实机验证为准。

### Event：单侧确认失联

- **接口**：`onLeftLinkLostEvent()` / `onRightLinkLostEvent()`
- **触发条件**：某一侧超过失联阈值并通过防抖确认，另一侧仍在线。
- **状态变化**：从在线 `SensorSet` 中移除失联侧。
- **反馈**：失联侧设备灯熄灭；剩余在线灯区继续显示距离；播放一次短下降音。
- **流程**：不等待；提示后按剩余在线侧的最新距离恢复蜂鸣。

### Event：双侧确认失联

- **接口**：`onBothLinksLostEvent()`
- **触发条件**：左右两侧均确认失联，或短时间内合并成双侧失联。
- **状态变化**：`SensorSet = None`，停止距离反馈。
- **反馈**：四灯进入系统不可用效果，播放一次合并故障短音。
- **流程**：退出工作模式并进入休眠或关机流程；不设计双侧失联后的自动恢复。

### Event：单侧恢复在线

- **接口**：`onLeftLinkRestoredEvent()` / `onRightLinkRestoredEvent()`
- **触发条件**：某一侧失联后连续收到有效数据，另一侧仍在线。
- **状态变化**：将恢复侧重新加入在线 `SensorSet`。
- **反馈**：恢复侧单灯绿色常亮，可播放短上升音；随后按最新距离恢复距离反馈。
- **流程**：不等待。

## 暂不设计

以下事件后续再补，不进入第一版核心 API：

- 主机普通低电
- 主机严重低电
- 严重系统故障
- 单侧距离数据持续异常
- 双侧距离数据持续异常
- 单侧设备普通低电
- 单侧设备严重低电或即将离线
- 工作模式主动退出或超时退出
- 测试模式、配置模式、恢复出厂模式
