# 雷达项目音效清单设计

## 命名和优先级规则

### 命名规则

| 项      | 规则                                                                                        |
| ------ | ----------------------------------------------------------------------------------------- |
| 文件格式   | 全部使用小写 ASCII、snake\_case、`.wav` 后缀。                                                       |
| 音频规格   | 统一 `16kHz / mono / 16-bit PCM WAV`。                                                       |
| 分类前缀   | `sys_` 系统，`mode_` 模式，`pair_` 配对，`wake_` 唤醒，`dist_` 距离，`link_` 通信，`power_` 电源，`fault_` 故障。 |
| 方向后缀   | 仅状态类音效在需要区分车外节点时使用 `_left`、`_right`、`_both`。                                              |
| 蜂鸣命名   | 倒车距离蜂鸣不区分左右，统一使用 `dist_beep_<level>.wav`。                                                 |
| 蜂鸣实现   | 蜂鸣节奏写进 WAV 文件，代码只负责 `playLoop()` 循环播放和分档切换。                                               |
| 语音提示   | 状态类按“语音优先”，但必须短促，避免长语音阻塞倒车实时蜂鸣。                                                           |
| 去重原则   | 同一事件链只播一个最高信息量音效。例如双侧失联优先于连续播放左失联和右失联。                                                    |
| 快流程原则  | 恢复出厂、未配对等快速流程只保留一个结果或状态提示，不设计“开始 + 完成”双提示。                                                |
| 预留模式原则 | 测试模式、配置模式不设计专用音效，避免误触发或与维护流程冲突。                                                           |
| 按键原则   | 不设计按键短确认和长按确认音效，避免与模式提示、配对提示、工作提示冲突。                                                      |

### 优先级规则

| 优先级     | 规则                                                                                            |
| ------- | --------------------------------------------------------------------------------------------- |
| P0      | 最高安全优先级：危险距离、严重故障、整套雷达不可用、即将关机等会直接影响倒车安全的事件，可打断普通提示。                                          |
| P1      | 倒车实时距离蜂鸣，循环播放，随距离等级切换；除 P0 外，应优先于普通状态语音。                                                      |
| P2      | 工作中重要状态：单侧失联、单侧唤醒失败、低电、通信异常等会降低系统可信度但不一定立即失去全部功能的事件。                                          |
| P3      | 模式/流程结果：配对结果、连接恢复、恢复出厂完成等需要明确反馈但不属于实时安全告警的事件。                                                 |
| P4      | 普通生命周期提示：启动、关机、进入模式、开始唤醒等低紧急度事件。                                                              |
| P0 内部仲裁 | 系统不可用类提示优先于危险距离蜂鸣：双侧通信失联、双侧传感器故障、双侧节点严重低电、本机严重低电、通用系统故障，应打断 `dist_beep_danger.wav` 并立即播报对应语音。 |

## 系统和模式音效

| 音效      | 触发场景                       | 优先级 | 文件名                           | 播报语音    |
| ------- | -------------------------- | --: | ----------------------------- | ------- |
| 启动提示    | 车内主机上电或按键唤醒后，扬声器初始化成功。     |  P4 | `sys_boot.wav`                | 系统已启动   |
| 进入休眠/关机 | 工作结束、超时或按键退出后准备进入深度睡眠。     |  P4 | `sys_shutdown.wav`            | 系统已关闭   |
| 未配对提示   | 用户短按进入工作，但本机未保存两个车外节点 MAC。 |  P3 | `mode_unpaired.wav`           | 设备未配对   |
| 进入配对模式  | 用户长按 USER 键进入无线配对模式。       |  P4 | `mode_pairing.wav`            | 进入配对模式  |
| 恢复出厂完成  | MAC、LoRa 配置清除完成后播一次。       |  P3 | `mode_factory_reset_done.wav` | 已恢复出厂设置 |

| 双侧唤醒失败   | 最大 LoRa 重试后左右节点均未唤醒。               |  P2 | `wake_fail_both.wav`              | 雷达未连接     |

| 双侧节点严重低电 | 左右车外节点均电量过低，倒车雷达即将不可用。             |  P0 | `power_sensor_critical_both.wav`  | 雷达电量过低    |

## 倒车距离音效

| 音效      | 触发场景                                 | 优先级 | 文件名                     | 播报语音    |
| ------- | ------------------------------------ | --: | ----------------------- | ------- |
| 距离安全/静音 | 最近有效距离 `>150cm` 或无报警需求。              |  P1 | 无文件                     | 无语音     |
| 远距蜂鸣    | `120cm < min_dist <= 150cm`，慢速循环蜂鸣。  |  P1 | `dist_beep_far.wav`     | 无语音，仅蜂鸣 |
| 中远距蜂鸣   | `90cm < min_dist <= 120cm`，中慢速循环蜂鸣。  |  P1 | `dist_beep_mid_far.wav` | 无语音，仅蜂鸣 |
| 中距蜂鸣    | `60cm < min_dist <= 90cm`，中速循环蜂鸣。    |  P1 | `dist_beep_mid.wav`     | 无语音，仅蜂鸣 |
| 近距蜂鸣    | `30cm < min_dist <= 60cm`，快速循环蜂鸣。    |  P1 | `dist_beep_near.wav`    | 无语音，仅蜂鸣 |
| 危险连续蜂鸣  | `0cm < min_dist <= 30cm`，连续或高急迫循环提示。 |  P0 | `dist_beep_danger.wav`  | 无语音，仅蜂鸣 |
